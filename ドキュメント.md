## プロジェクト概要

**DARK FOREST** は Three.js + Vite で構築された3Dブラウザホラーゲームです。

- **エンジン**: Three.js (WebGL 3Dレンダリング)
- **ビルド**: Vite (高速バンドラー)
- **通信**: WebSocket (`ws` ライブラリ)
- **音声**: Web Audio API (手続き生成)
- **難読化**: javascript-obfuscator

### ゲーム内容
- プロシージャル生成された暗い森でプレイ
- 光の欠片を3つ集めてポータルで脱出
- 3体の敵 (Ember, Dester, Runder) から逃げながら進む
- 最大4人のオンラインマルチプレイ対応

## 起動方法

```bash

```

## アーキテクチャ

### ゲームループ (Game.js)
```
Game.constructor()
  ├── _initRenderer()     → WebGLRenderer設定
  ├── _initScene()        → Scene, Camera, Controls
  ├── _setupEventListeners() → キーボード, ポインターロック
  └── _animate()          → requestAnimationFrame ループ
        ├── Player.update()
        ├── Flashlight.update()
        ├── World.update()
        ├── ItemSystem.update()
        ├── Enemy[].update()  (ホストのみ)
        ├── NetworkSync       (マルチ時20Hz)
        ├── AudioManager.updateDissonance()
        └── HUD/Minimap更新
```

### 状態マシン
```
MENU → PLAYING ⇄ PAUSED
         ↓
  GAMEOVER / VICTORY
```

### 敵AI状態マシン (EnemyBase.js)
```
PATROL → CHASE → ATTACK
   ↑       ↓
   ← EXHAUSTED
   
30秒プレイヤー未発見 → テレポート
```

### マルチプレイヤーアーキテクチャ
```
[Host Client]  ←→  [WebSocket Server]  ←→  [Guest Client ×1~3]
  - マップ生成         - ルーム管理           - 位置送信
  - 敵AI実行          - メッセージリレー      - 状態受信
  - 状態配信          - パスワード検証        - 描画のみ
```

## 各モジュール詳細

### Game.js (メインオーケストレーター)
- **役割**: 全システムの初期化・更新・連携
- **主要メソッド**:
  - `_startGame(multiplayer)` - ゲーム初期化
  - `_animate()` - 毎フレーム更新ループ
  - `_updateEnemies(dt)` - 敵AI更新 (ホストのみ)
  - `_sendGameState()` - ネットワーク状態同期

### Player.js
- **Player**: FPS操作、衝突判定、スタミナ管理
- **RemotePlayer**: 他プレイヤーのメッシュ・名前タグ・位置補間

### World.js (パフォーマンス最適化済み)
- `InstancedMesh` で500本の木を2ドローコールで描画
- 地面、アンダーグロウス、ホタル、小屋を生成

### EnemyBase.js
- 状態マシン: PATROL / CHASE / ATTACK / EXHAUSTED
- テレポート: 30秒未発見でプレイヤー60-120m先にワープ
- 木との衝突回避付きパスファインディング

### AudioManager.js
- Web Audio API でリアルタイム音声生成
- 不協和音: 敵が近づくと自動的に強くなる
- ジャンプスケア、足音、アイテム効果音

### NetworkManager.js
- WebSocket接続管理
- イベントベースのメッセージハンドリング
- ルーム操作 (作成/参加/退出)

## マルチプレイヤー通信

### メッセージプロトコル

| メッセージ | 方向 | 説明 |
|-----------|------|------|
| SET_NAME | C→S | プレイヤー名設定 |
| CREATE_ROOM | C→S | ルーム作成 (パスワード付き) |
| JOIN_ROOM | C→S | ルーム参加 |
| ROOM_INFO | S→C | ルーム情報更新 |
| START_GAME | Host→All | ゲーム開始+マップデータ |
| PLAYER_UPDATE | C→Host | プレイヤー位置/HP (20Hz) |
| GAME_STATE | Host→Guests | 敵位置+全プレイヤー状態 |
| CHAT | C→All | チャットメッセージ |
| PLAYER_DIED | C→All | プレイヤー死亡通知 |
| GAME_OVER | Host→All | 全滅通知 |
| GAME_CLEAR | Host→All | 全員脱出成功 |

### クリア条件
- **ソロ**: ポータルに入れば即クリア
- **マルチ**: 全生存プレイヤーがポータルに入ったらクリア
- **ゲームオーバー**: 全プレイヤー死亡時

### スペクテイター
- 死亡するとスペクテイターモードに移行
- フリーフライカメラで自由に見回せる
- 画面が明るくなり見やすくなる

## 難読化とビルド

### 開発モード (難読化なし)
```bash
npm run dev
```
ソースコードがそのまま読める状態。デバッグ用。

### 本番ビルド (難読化あり)
```bash
npm run build
```
1. Vite がバンドル・ミニファイ → `dist/`
2. `scripts/obfuscate.js` が追加難読化を実行

### 難読化で行われること
- **変数名変換**: 全変数名を16進数に変換
- **文字列暗号化**: 文字列をBase64 + 配列化
- **制御フロー平坦化**: if/switch文を解読困難に
- **デッドコード挿入**: ダミーコードを混入
- **コンソール無効化**: console.log等を無効化
- **セルフディフェンス**: コード整形すると動作停止

### 難読化の解除 (開発者向け)
本番ビルドの難読化を解除するには:
1. `dist/` を削除
2. `npm run build:clean` (難読化なしビルド)
3. または `npm run dev` で開発モードを使用

ソースコード自体は `src/` に常に平文で存在します。
難読化は `dist/` の配布用ビルドのみに適用されます。

## 他県の人と遊ぶ方法

### 方法1: ngrok (最も簡単)
```bash
# 1. ngrokをインストール (https://ngrok.com)
# 2. WebSocketサーバーを起動
npm run server

# 3. 別ターミナルでngrokトンネル作成
ngrok tcp 8080

# 4. 表示されたURL (例: tcp://0.tcp.ngrok.io:12345) を
#    ゲーム内の接続先に入力
```

### 方法2: ポート開放
1. ルーター設定でポート 8080 をTCP開放
2. パブリックIPを確認 (https://whatismyip.com)
3. 相手に `ws://あなたのIP:8080` を共有

### 接続先の変更
`src/network/NetworkManager.js` の `connect()` メソッド:
```javascript
// デフォルト: ws://localhost:8080
// 変更例: ws://0.tcp.ngrok.io:12345
connect(url = 'ws://localhost:8080')
```
